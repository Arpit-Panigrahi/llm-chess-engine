# Experiment Protocol: Constrained LLM Chess Move Selection

This document outlines the standard protocol for running chess telemetry experiments with Llama-3 move selection under varied temperatures and constraint regimes.

## Objective

To measure the effect of sampling temperature and token-level constraints (injecting valid move choices in prompt context) on the rate of legal move generation by an LLM playing chess.

## Experiment Configuration Matrix

Every standardized experiment consists of running three distinct conditions, each parameterized to isolate the impact of variables.

| Condition Tag | Temperature | Constrained Decoding | Purpose |
|:---|:---:|:---:|:---|
| `t02_unconstrained` | 0.2 | No | Baseline low-temperature move generation without prompt injection. |
| `t08_unconstrained` | 0.8 | No | Baseline mid-temperature move generation without prompt injection. |
| `t08_constrained` | 0.8 | Yes | Test variable. Measures constraint efficacy under identical temperature. |

## Experimental Controls & Fairness

To ensure pairwise comparisons are valid, all run configurations MUST hold the following variables constant across the entire matrix:

1. **Model Version**: The same model (default: `llama3` 8B parameter model via Ollama) must be used.
2. **Deterministic Seed**: The same base random seed (default: `42`) must be passed to Ollama (`options.seed`) and the Python random move selector to ensure identical gameplay branch decisions.
3. **Dataset (Opponent)**: White plays random legal moves generated by the same pseudo-random generator state (based on game number + seed).
4. **Game Volume**: Exactly `50` games (or an identical count) must be run per condition to ensure statistically comparable sample sizes.
5. **Turn Cap**: Games are limited to `200` ply (100 full moves) to prevent infinite loops from repetitive moves.

## Execution Procedure

Experiments are orchestrated using the automated matrix script. To run a standard evaluation:

```bash
# Execute standard matrix (50 games/condition, seed 42)
bash scripts/run_experiment_matrix.sh
```

To run with custom volume or seed:

```bash
bash scripts/run_experiment_matrix.sh --num-games 100 --seed 123
```

## Telemetry Aggregation

The matrix runner generates artifacts under a structured subdirectory in `runs/` for each condition:
- `manifest.json`: Configuration, metadata, and high-level summaries.
- `metrics.json`: Evaluated rates and latencies.
- `raw_outputs.jsonl`: Line-by-line turns logging FEN, prompt constraints, latency, raw Ollama response, and legality status.

These are then fed into the analysis script (`scripts/analyze_all.py`) to construct the final comparative report.
