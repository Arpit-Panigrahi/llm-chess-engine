# Results and Telemetry Artifact Schema

This document details the file structures and column definitions for all primary outputs generated by the chess engine, tournament GUI, and experiment matrix.

---

## 1. Raw Game Outputs (`raw_outputs.jsonl`)

Generated incrementally during games played via `scripts/run_game.py`. Recorded under `runs/<RUN_ID>/raw_outputs.jsonl`. Each line is a single JSON object.

### Fields

| Field | Type | Description |
|:---|:---|:---|
| `game_id` | `int` | Sequential ID of the game within this run. |
| `turn_number` | `int` | Incremental ply number (1-based half-move count). |
| `timestamp` | `str` | ISO-8601 UTC timestamp of the move decision. |
| `fen` | `str` | Forsyth-Edwards Notation of the board prior to the move. |
| `temperature` | `float` | Sampling temperature option used for the Ollama model. |
| `constrained_decoding` | `bool` | Flag designating whether legal moves were injected into the prompt. |
| `latency_ms` | `int` | Round-trip request time to the Ollama endpoint. |
| `extracted_move` | `str` | Clean UCI-formatted move string extracted from raw response. |
| `played_move` | `str` | Move executed (same as extracted if legal, otherwise the random fallback). |
| `is_legal` | `int` | `1` if the extracted move is legal; `0` if illegal/empty. |
| `fallback_used` | `int` | `1` if random fallback played; `0` if LLM move played. |
| `raw_response` | `str` | Unprocessed raw text returned by the LLM api. |
| `num_legal_moves` | `int` | Total count of valid UCI choices in this position. |

---

## 2. Resolved Configuration (`config.resolved.json`)

Saved at run initiation at `runs/<RUN_ID>/config.resolved.json`. Dictates parameter replication.

### Fields

| Field | Type | Description |
|:---|:---|:---|
| `temperature` | `float` | LLM temperature value. |
| `constrained_decoding` | `bool` | Legal constraint active flag. |
| `seed` | `int` | RNG seed for deterministic evaluation. |
| `model` | `str` | Target Ollama model name. |
| `ollama_base_url` | `str` | Destination URL of the Ollama server. |
| `num_games` | `int` | Total target games count. |
| `tag` | `str` | Identifier tag (e.g., `t02_unconstrained`). |
| `run_id` | `str` | Unique, chronological Run ID. |
| `time_limit` | `float` | Request timeout per move in seconds. |

---

## 3. Run Manifest (`manifest.json`)

Metadata and high-level summaries. Saved at `runs/<RUN_ID>/manifest.json`.

### Fields

- `schema_version`: Version signature (currently `"1.0"`).
- `run_id`: Run identifier.
- `tag`: Optional condition tag.
- `started_at`: UTC start timestamp.
- `completed_at`: UTC end timestamp.
- `ollama_mode`: Detected host network topology environment (`local`, `docker`, `wsl`, or `unknown`).
- `config`: Complete copy of the input parameter config.
- `summary`:
  - `total_games`: Count of games played.
  - `legal_move_rate`: Percentage of valid moves chosen by LLM.
  - `unique_moves`: Counts distinct extracted move types.
  - `latency_mean_ms`: Mean latency.

---

## 4. Run Metrics (`metrics.json`)

Evaluated statistical aggregates of the run. Saved at `runs/<RUN_ID>/metrics.json`.

- `schema_version`: `"1.0"`
- `run_id`: Unique identifier.
- `tag`: Condition label.
- `condition`: Summary of active parameters.
- `total_games`: Matches num_games.
- `total_llm_calls`: Count of turns Black played.
- `total_legal_moves`: Count where `is_legal == 1`.
- `total_fallback_moves`: Count where fallback was triggered.
- `legal_move_rate`: Calculated ratio between `0.0` and `1.0`.
- `unique_moves`: Distinct set of legal or illegal UCI moves proposed.
- `latency_mean_ms` / `latency_median_ms` / `latency_min_ms` / `latency_max_ms`: Latency aggregates mapping.
- `game_results`: List of JSON objects mapping outcome and length of each game.

---

## 5. Comparative CSV (`metrics_comparison.csv`)

Generated in the output folder by the analysis script. Aggregates the `metrics.json` records of compared conditions.

### Columns
`tag`, `run_id`, `temperature`, `constrained`, `seed`, `model`, `total_games`, `total_llm_calls`, `legal_moves`, `legal_rate`, `unique_moves`, `latency_mean_ms`, `latency_median_ms`.
